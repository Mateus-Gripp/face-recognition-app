FaceID Lab

Sistema Web de Reconhecimento Facial com Prova de Vida

Projeto full-stack construído para transformar conceitos de visão computacional e processamento de imagens em um sistema web funcional de cadastro e identificação facial.

Foco da apresentação: o que foi prometido, o que foi entregue e como a visão computacional aparece no sistema

⚙️ Backend Node.js + Express + Prisma
🗄️ Base SQLite
🧠 IA Facial @vladmandic/face-api
1. Ideia do projeto

O que eu quis construir com o FaceID Lab

Criar um sistema web prático que usa prova de vida e visão computacional para cadastro e identificação facial.

Objetivo

  • criar um sistema web de cadastro e identificação facial com aplicação prática
  • usar prova de vida para reduzir fraudes simples, como foto estática e cadastro duplicado
  • aplicar, de forma visível, conceitos estudados em PDI e visão computacional

Fluxo central

A proposta foi transformar conceitos de PDI em uma aplicação capaz de capturar, analisar e reconhecer rostos, com interface utilizável e uma camada básica de segurança.

2. Como o sistema funciona

Como eu organizei a aplicação

A parte mais importante do fluxo é esta: a webcam captura, o frontend envia a imagem para a API e a API decide o que fazer com aquele rosto.

🖥️ Frontend
  • captura por webcam
  • páginas de cadastro e identificação
  • feedback visual para o usuário
⚙️ Backend
  • API REST
  • validação dos dados
  • comparação e regras de segurança
🗄️ Base de dados
  • pessoas cadastradas
  • descritores faciais
  • imagens e metadados
Webcam → Envio para a API → Detecção e recorte do rosto → Descritor facial → Validação e comparação
3. Processamento na API

O que acontece quando a imagem chega no servidor

Leitura simples do processamento

1

A API recebe a foto capturada e confere se há um rosto utilizável na imagem.

2

Depois disso, o sistema isola a região do rosto para trabalhar menos com o fundo e mais com a face.

3

Com esse rosto, a biblioteca gera um resumo numérico da face, que funciona como uma assinatura facial.

4

Por fim, a API compara essa assinatura com a base e decide se cadastra, bloqueia duplicidade ou identifica alguém.

Onde entram os conceitos da disciplina

  • a imagem da webcam é a etapa de aquisição
  • a face detectada vira a região de interesse
  • o descritor representa a extração de características
  • a distância entre descritores guia a decisão final
  • a API transforma esses passos em uma resposta prática para o sistema

Em termos simples, a API não "entende" a foto como uma pessoa: ela transforma a imagem em características comparáveis e usa isso para tomar a decisão.
